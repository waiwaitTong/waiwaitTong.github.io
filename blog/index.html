动机

经学长建议决定搭一个个人网站 以记录自己各种 可以避免出现想发表重要意见 但没有具体的发表对象 又不乐意发朋友圈的情况

收获

  1. Markdown使用

  2. AI辅助与认清AI

步骤

环境准备​

  1. 生成 SSH 密钥：ssh-keygen -t rsa -C "your_email@example.com"

  2. 将公钥添加到 GitHub：cat ~/.ssh/id_rsa.pub

  3. 配置 SSH 使用 443 端口

项目部署​

# 1. 初始化 Git 仓库
git init
git add .
git commit -m "Initial commit"

# 2. 创建 GitHub 仓库（空仓库）
# 3. 设置远程仓库
git remote add origin git@github.com:waiwaitTong/waiwaitTong.github.io.git
git push -u origin master

# 4. 创建 gh-pages 分支
git checkout --orphan gh-pages
echo "# GitHub Pages" > README.md
git add README.md
git commit -m "Initial gh-pages"
git push origin gh-pages
git checkout master

# 5. 确认 baseUrl 配置正确
# 在 docusaurus.config.js 中设置 baseUrl: '/'

# 6. 部署
yarn deploy

问题及解决方案

环境配置问题​

cmd检测版本号成功 powershell不成功

尝试重装后 开开关关cmd和powershell 再检测又成功

deepseek解释说下载前打开终端导致

那以后多开开关关终端好了

权限问题​

管理员权限运行即可

或一劳永逸：将code文件夹属性安全改为完全控制

网页不刷新​

并非不刷新

只是新手初始化教程中选择的页面问题

网页崩溃​

不知道啥情况 直接重启网站好了

编辑后打开巨卡无比​

npm run build重启

GitHub Pages 部署问题总结​

以为成功后网站简陋且丢失blog页面​

似乎是github强制使用自己的网站部署

使用deepseek和csdn推送的教程均不顶用

于是选择一切初始化 重新连接仓库开始

https://www.docusaurus.io/docs/deployment#deploying-to-github-pages事实上有官方文档

可读性不强

让AI总结了一份

SSH 密钥生成与配置问题​

ssh: connect to host github.com port 22: Connection refused
# 使用 443 端口测试连接
ssh -T -p 443 git@ssh.github.com

# 创建 SSH 配置文件
mkdir -p ~/.ssh
cat > ~/.ssh/config << 'EOF'
Host github.com
    Hostname ssh.github.com
    Port 443
    User git
EOF

# 设置文件权限
chmod 600 ~/.ssh/config
